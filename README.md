# Remnawave Routing Auto Updater

Автономный микросервис на Python + Docker для автоматического обновления баз роутинга (`geoip`, `geosite`) для клиентов **Happ** и **INCY** в панели управления [Remnawave](https://github.com/remnawave).

Скрипт использует **официальный API Remnawave** для обновления настроек подписки, что делает его максимально безопасным (никаких доступов к базе данных напрямую!). Логирование автоматически ротируется встроенными средствами Docker.

## Особенности
* Легковесный образ на базе `Alpine Linux` (~30 МБ).
* Работа через официальный API Remnawave (по токену).
* Поддержка как локального запуска (в одной сети с панелью), так и удаленного.
* Автоматическая генерация `routing.json` для `autorouting` INCY.

## ⚙️ Требования и логика работы
* **Для клиентов HAPP:** Скрипт работает "из коробки". Он автоматически обновляет стандартное поле `Happ routing` в настройках подписки. Если у вас создано кастомное правило в Custom Headers с именем `Happ`, скрипт обновит и его.
* **Для клиентов INCY:** Вам необходимо зайти в Remnawave → **Раздел "Подписка"** → **Правила ответов**, создать правило с именем `Incy` и добавить заглушки для ключей `routing` и `autorouting` (в значение можно вписать любой текст, скрипт заменит его на актуальные ссылки).
* В разделе **Настройки Remnawave** → **API токены** создайте новый токен (Права Read/Write в разделе Subscription Template) и скопируйте его в файл `.env`.

## Установка и запуск

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/YOUR_GITHUB_NAME/remnawave-routing-updater.git
   cd remnawave-routing-updater
   ```

2. Настройте конфигурацию:
   ```bash
   cp .env.example .env
   nano .env
   ```
   *Вставьте ваш `API_TOKEN`. Если запускаете скрипт локально (на том же сервере), оставьте `PANEL_URL=http://remnawave:3000`.*

3. Отредактируйте `template.json`, добавив свои правила (DirectSites, базы обхода и т.д.). **Внимание:** Поле `LastUpdated` добавлять не нужно, скрипт генерирует его автоматически.

4. **[Опционально]** Если скрипт работает на том же сервере, раскомментируйте блоки `networks` в файле `docker-compose.yml`, чтобы подключить его к внутренней сети панели.

5. Запустите сервис:
   ```bash
   docker-compose up -d --build
   ```

6. Просмотр логов:
   ```bash
   docker logs -f remna-routing-updater
   ```

## Настройка веб-сервера (Nginx) для INCY
Для работы `autorouting` клиент INCY должен скачивать `routing.json` по прямой ссылке. Скрипт генерирует этот файл в папку `./output`. Добавьте алиас в конфигурацию вашего домена:

```nginx
# Вставьте этот блок перед 'location /'
location /routing.json {
    alias /полный/путь/к/папке/remnawave-routing-updater/output/routing.json;
    add_header Content-Type application/json;
}
```
Перезапустите Nginx: `systemctl reload nginx`
